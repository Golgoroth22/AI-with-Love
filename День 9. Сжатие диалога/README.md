#  День 8. Работа с токенами

1. Добавьте в код подсчёт токенов (на запрос и ответ)
2. Сравните: короткий запрос, длинный запрос и запрос, превышающий лимит модели

- `Результат`: Код, который считает токены и показывает, как меняется поведение модели
- `Формат`: Видео + Код

[Видео результата](https://drive.google.com/file/d/11UQy07O_i_GATxT4w877l2oGMAhVM-vR/view?usp=sharing)
### Вывод
- Короткий запрос даёт быстрый, но общий и менее точный ответ, модель больше "додумывает"
- Длинный запрос в пределах лимита обычно даёт более специфичный, связный и полезный ответ, но при лишней "воде" качество может падать
- Запрос, превышающий лимит токенов, приводит к ошибке либо к усечению части текста, из‑за чего модель теряет контекст и начинает отвечать менее корректно
